Halo reads your real Ghostty config, so any Ghostty option works — plus a handful of halo- keys for the chrome. Drive it from scripts and agents over a local control socket.

Halo config keys

Put these in your Ghostty config (or Halo's own config after Settings ▸ Import). libghostty ignores the halo- prefix, so they pass straight through.

KeyDefaultWhat it does
halo-projects—Comma-separated project paths for the sidebar (~/a, ~/b).
halo-accenttheme accentSelection, focus ring, and active-session color.
halo-surfaceghostty backgroundWindow + pane background override.
halo-sidebar-width224Sidebar width, in points.
halo-font-familyGeistMonoSidebar / UI text family.
halo-font-monoMartianMonoInstrument-label family (uppercase chips).
halo-font-size13Base UI font size.
halo-divider-width8Split-divider grab width.
halo-prefixctrl+btmux-style prefix key. Empty value disables prefix mode.
halo-prefix-bind—Rebind a prefix key: <key>:<action>, comma-separated for multiple, e.g. v:split-vertical. Actions: split-vertical, split-horizontal, focus-left/down/up/right, zoom, new-session, next-session, prev-session, rename, detach, kill.
halo-persistonRun sessions through the background daemon so shells survive Halo quitting. Set false for plain in-process shells (no persistence).

Inherited Ghostty keys

Anything in the Ghostty config reference works in Halo. Colors, fonts, and themes sync automatically. A few common ones:

themeNamed theme, e.g. theme = "Atom" (Halo bundles all 463).
font-familyTerminal font, e.g. font-family = "Redaction 50".
background / foregroundBase colors, e.g. #161719.
cursor-colorCursor color.
window-padding-x / -yInner padding around the grid.

Change anything, then Settings ▸ Reload (or halo reload) — no relaunch.

Sessions & persistence

Sessions run under a small background daemon (halod). Shells survive Halo quitting or crashing — and survive closing a window — then reattach on next launch with native scrollback restored. Long scrollback spills to disk, so history is recovered even if the daemon restarts; inline images replay across reattach.

The sidebar is the single source of truth: every session lives there. Closing a window just closes a view — the sessions stay alive and reopen with the window. Closing a session (its last pane) kills its shell, so there are no hidden orphans.

Open more than one window with ⌘N; they all share the same sidebar and session pool. Each window views its own session, so you can have a different one live in each. If two windows point at the same session, the focused window shows it live and the other shows a muted frozen snapshot — focus a window and the live view follows.

Agent control CLI

The halo command drives and observes the running app over a unix socket (~/Library/Application Support/halo/control.sock, owner-only) — built so a script or an AI agent can run commands and read back what happened. Run bare halo while the app is open and it opens a new window in the running instance.

# open an isolated worktree session, split it, run tests, read the output
halo worktree fix/login
halo split -h
halo send-keys focused "npm test\n"
halo capture --scrollback

Lua scripting

Halo embeds Lua 5.4. On launch (and on halo reload) it runs ~/.config/halo/init.lua with a global halo table. A script error is shown as a toast — it never crashes the app.

-- ~/.config/halo/init.lua

-- toast when a command/agent turn finishes in any session
halo.on("command-finished", function(id) halo.notify("done: "..id) end)

-- ctrl+g → git status in the focused pane
halo.bind("ctrl+g", function() halo.send("git status\n") end)

-- a command runnable with `halo run tests` or from a keybind
halo.command("tests", function()
  local p = halo.active()
  halo.notify("running tests in "..p.cwd)
  halo.send("npm test\n")
end)
APIWhat it does
halo.notify(msg)Show a toast banner in the active window.
halo.command(name, fn)Register a command, runnable via halo run <name> or a keybind.
halo.bind(chord, fn)Bind a key chord (e.g. "cmd+shift+p", "ctrl+g") to a Lua function.
halo.on(event, fn)Run fn on an event (below). The handler gets the session's paneID where relevant.
halo.send(text)Type text into the focused pane (include "\n" to run it).
halo.active()Table for the focused session: {cwd, title, paneID}.

Events

session-openedA session/tab was created (arg: paneID).
session-closedA session was closed (arg: paneID).
focus-changedThe active session changed (arg: paneID).
command-finishedA foreground command/agent turn returned to the shell (arg: paneID).
config-reloadedinit.lua (re)loaded — fires once after load and on every halo reload.

Plugins

Plugins are Lua packages that use the same halo API. Declare them in init.lua and Halo clones them into ~/.config/halo/plugins/ on first launch; or just drop a folder there yourself. Each plugin's init.lua (or plugin/init.lua) runs at startup and registers its own commands, events, and keybinds.

-- ~/.config/halo/init.lua
halo.plugin("you/halo-statusbar")   -- github.com/you/halo-statusbar
halo.plugin("you/halo-fzf")

Manage them from the CLI: halo plugins lists what's installed, and halo plugins sync git pulls each one and reloads. A drop-in folder needs no declaration — ~/.config/halo/plugins/my-thing/init.lua just works.
